# Step 01 — Ideation

## Input
User gives a topic, angle, or rough idea for a carousel.

## Process

1. **Read brand config** — `resources/brand.md` for language, tone, audience pain points
2. **Read context** — Scan `context/` for relevant docs (if specified in brand.md)
3. **Identify content level** — TOFU (50%), MOFU (30%), or BOFU (20%)
4. **Propose 3 angles** — Each with:
   - Hook title (what slide 1 would say — in the brand language)
   - Content level (TOFU/MOFU/BOFU)
   - Why this angle works for the target audience
   - Estimated slide count (5-10)
5. **User picks an angle** (or mixes)
6. **Propose carousel structure** — One line per slide:
   ```
   Slide 1 [Hook]: "Your hook headline here"
   Slide 2 [Content]: The shift — old way vs. new way
   Slide 3 [Content]: The core concept explained
   ...
   Slide 7 [CTA]: Save this for later
   ```
7. **User validates** the structure → move to Step 02

## Output
Validated carousel structure with slide count and one-liner per slide.
